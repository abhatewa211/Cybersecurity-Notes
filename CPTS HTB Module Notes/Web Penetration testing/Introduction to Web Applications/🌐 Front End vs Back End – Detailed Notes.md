## 🔑 Definition:

- The **front end** contains everything the **user sees and interacts with** in the browser.
    

### 🧱 Core Technologies:

- **HTML** → Structure
    
- **CSS** → Design & styling
    
- **JavaScript** → Functionality
    

---

## 🎯 Responsibilities:

- Display:
    
    - Text
        
    - Images
        
    - Buttons
        
- Handle user interaction
    
- Execute code in browser (real-time)
    

---

## 📱 Modern Front-End Requirements:

- Responsive design (mobile + desktop)
    
- Works on all browsers
    
- Fast and optimized
    

⚠️ Poor front-end optimization →

- Slow website
    
- Bad user experience
    
- Misleading performance issues
    

---

## 🎨 Front-End Development Tasks:

- UI (User Interface) design
    
- UX (User Experience) design
    
- Visual web design

## 🔑 Definition:

- The **back end** handles all **core logic and processing** of the web application.
    

✔ Users do NOT directly see it

---

## 🧩 Main Back-End Components:

|Component|Description|
|---|---|
|**Back-End Server**|Hardware + OS (Linux/Windows)|
|**Web Server**|Handles HTTP (Apache, NGINX, IIS)|
|**Database**|Stores data (MySQL, MongoDB, etc.)|
|**Frameworks**|App logic (Django, Laravel, Spring, Express)|

---

## ⚙️ Back-End Responsibilities:

- Process user requests
    
- Implement application logic
    
- Manage database
    
- Build APIs
    
- Integrate services (cloud, external APIs)
    

---

## 🐳 Deployment & Isolation:

- Use **Docker containers** for isolation
    
- Separate:
    
    - App
        
    - Database
        
    - Services
        

✔ Improves security & stability

- Even without back-end access → app can be attacked

## 🔥 Common Attacks:

### 1. SQL Injection

- Manipulate queries
    
- Access database
    

### 2. Command Injection

- Execute OS commands
    

---

## 🧪 Testing Types:

### 🟢 Whitebox Pentesting

- Full source code access
    
- Code review possible
    

### ⚫ Blackbox Pentesting

- No source code
    
- Test externally
    

---

## 🔓 Special Cases:

- Open-source apps → code available
    
- File Inclusion → leak source code

(VERY IMPORTANT – Keep as-is)

1. Permitting Invalid Data to Enter the Database
    
2. Focusing on the System as a Whole
    
3. Establishing Personally Developed Security Methods
    
4. Treating Security to be Your Last Step
    
5. Developing Plain Text Password Storage
    
6. Creating Weak Passwords
    
7. Storing Unencrypted Data in the Database
    
8. Depending Excessively on the Client Side
    
9. Being Too Optimistic
    
10. Permitting Variables via the URL Path Name
    
11. Trusting third-party code
    
12. Hard-coding backdoor accounts
    
13. Unverified SQL injections
    
14. Remote file inclusions
    
15. Insecure data handling
    
16. Failing to encrypt data properly
    
17. Not using a secure cryptographic system
    
18. Ignoring layer 8
    
19. Review user actions
    
20. Web Application Firewall misconfigurations

# 🧠 Key Takeaways

- Front end = **User interaction layer**
    
- Back end = **Core logic + data processing**
    
- Both must be **secured properly**
    
- Many attacks target **input handling flaws**
    
- Understanding both sides is critical for **pentesting**
