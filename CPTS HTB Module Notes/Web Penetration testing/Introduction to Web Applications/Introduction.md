- A **web application** is an **interactive application** that runs in a **web browser**.
    
- Uses a **client-server architecture**:
    
    - **Front-end (Client-side)** → What the user sees (HTML, CSS, JS)
        
    - **Back-end (Server-side)** → Logic, database, processing
        

### 🔑 Examples:

- Gmail
    
- Amazon
    
- Google Docs
    

✔ Accessible worldwide  
✔ Real-time interaction  
✔ Hosted on servers

## 🛡️ Web Application Penetration Testing

- Critical for securing applications
    
- Must test:
    
    - Front-end
        
    - Back-end
        
    - Authentication systems
        

### 🔍 Testing Approach:

1. Analyze **front-end (HTML, CSS, JS)**
    
2. Identify vulnerabilities:
    
    - XSS
        
    - Sensitive Data Exposure
        
3. Test **server interaction**
    
4. Test both:
    
    - Unauthenticated access
        
    - Authenticated access
        

### 📘 Standard:

- OWASP Web Security Testing Guide

- Web apps provide a **large attack surface**

### 🔥 Common Attacks:

#### 1. SQL Injection

- Exploits unsafe input handling
    
- Can:
    
    - Access database
        
    - Read/write files
        
    - Execute code
        

#### 2. File Inclusion

- Read sensitive files
    
- Discover hidden paths
    

#### 3. Unrestricted File Upload

- Upload malicious files
    
- Gain server control
    

#### 4. IDOR (Insecure Direct Object Reference)

- Access other users’ data
    
- Example:
    
    ```
    /user/701 → change to /702
    ```
    

#### 5. Broken Access Control

- Privilege escalation
    
- Example:
    
    ```
    roleid=3 → change to roleid=1 (admin)
    ```

## 🔗 Attack Chaining Concept

- One vulnerability → leads to another
    
- Example:
    
    - SQL Injection → get emails
        
    - Emails → password spraying
        
    - Gain system access
        

✔ Real-world attacks often use **multiple chained vulnerabilities**
