- Archive files are used to **combine multiple files into one** for:
    
    - Easy sharing
        
    - Better organization
        
    - Reduced size (compression)
        
- Common formats:
    
    - `.zip`, `.rar`, `.tar`, `.gz`, `.7z`, `.gzip`
        
    - `.vmdb`, `.kdbx`, `.deb`, `.bitlocker`
        
- ⚠️ Important:
    
    - Not all archive formats support passwords natively.
        
    - Extra encryption tools like **OpenSSL** or **GPG** may be used.

## 🔑 Key Concept: Password Cracking Workflow

1. Extract hash from file
    
2. Use wordlist (e.g., `rockyou.txt`)
    
3. Run cracking tool
    
4. Recover password

### 📌 Steps:

### 1. Extract Hash

```bash
zip2john ZIP.zip > zip.hash
```

### 2. Crack Password

```bash
john --wordlist=rockyou.txt zip.hash
```

### 3. Show Results

```bash
john zip.hash --show
```

### 📌 Example Output:

```
ZIP.zip/customers.csv:1234
```

✔️ Password = `1234`

---

## 🧠 Important Notes

- ZIP cracking uses **PKZIP hash format**
    
- Works offline (no internet needed)
    
- Speed depends on:
    
    - CPU power
        
    - Wordlist quality

### 🔐 What is BitLocker?

- Full disk encryption (Windows)
    
- Uses **AES-128 / AES-256**
    
- Can unlock via:
    
    - Password
        
    - 48-digit recovery key
        

---

## 📌 Extract Hashes

```bash
bitlocker2john -i Backup.vhd > backup.hashes
```

Filter password hash:

```bash
grep "bitlocker$0" backup.hashes > backup.hash
```

---

## ⚡ Crack Using Hashcat

```bash
hashcat -a 0 -m 22100 backup.hash rockyou.txt
```

### 📌 Example Result:

```
Password: 1234qwer
```

---

## ⚠️ Important Points:

- Mode `22100` = BitLocker
    
- Very **slow cracking** (strong encryption)
    
- Recovery key is usually **impractical to brute-force**

## 🐧 On Linux / macOS

### Install Tool:

```bash
sudo apt-get install dislocker
```

---

### Setup Mounting:

```bash
sudo mkdir -p /media/bitlocker
sudo mkdir -p /media/bitlockermount
```

---

### Mount Process:

```bash
sudo losetup -f -P Backup.vhd
sudo dislocker /dev/loop0p2 -uPASSWORD -- /media/bitlocker
sudo mount -o loop /media/bitlocker/dislocker-file /media/bitlockermount
```

---

### Unmount:

```bash
sudo umount /media/bitlockermount
sudo umount /media/bitlocker
```

# 🧾 Key Takeaways

- Archive cracking depends on **format + encryption method**
    
- Tools used:
    
    - `zip2john` → ZIP
        
    - `john` → cracking
        
    - `hashcat` → fast GPU cracking
        
    - `openssl` → encrypted gzip
        
    - `dislocker` → mount BitLocker
        

---

## ⚠️ Important Security Insight

- Weak passwords = easily cracked
    
- Strong encryption (like BitLocker) = time-consuming
    
- Always use:
    
    - Long passwords
        
    - Mixed characters
        
    - Secure storage
